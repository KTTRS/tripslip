# Field Trip Platform — Replit Build Spec

## Objective

Build an MVP that discovers field trip venues near a school, enriches venue data, extracts educational programs from venue websites, ranks results, and exposes a searchable API.

The system should be optimized for:

* nearby field trip discovery
* structured venue/program records
* image support where available
* deterministic extraction first, LLM normalization second
* auditable evidence for extracted program fields

---

### Supported venue types

* museum

## System architecture

```text
School Input
  -> Geocoder
  -> Venue Discovery Engine
  -> Venue Normalization + Deduplication
  -> Venue Enrichment
  -> Program Extraction Pipeline
  -> Recommendation Engine
  -> Search API
  -> Frontend / Admin UI
```

---

## Core user flow

1. User enters school address or selects school.
2. System geocodes the school.
3. System queries nearby POIs within a radius.
4. Results are normalized into supported venue types.
5. Candidate venues are deduplicated and stored.
6. Venue websites are crawled for education / field trip pages.
7. Program data is extracted.
8. Results are ranked by distance + grade fit + subject match + data quality.
9. API returns venue cards with image, distance, and program summaries.

## Geoapify discovery flow

### Step 1: geocode school

Use school lat/lon already stored if available.

### Step 2: query nearby POIs by category bundles

Do not do one giant query for all categories. Query by bundle and merge.

Suggested bundles:

* museums
* zoos/aquariums
* nature/parks
* historic sites
* farms

### Example request shape

```python
import httpx

async def geoapify_places(lat: float, lon: float, radius_meters: int, categories: list[str], api_key: str):
    url = "https://api.geoapify.com/v2/places"
    params = {
        "categories": ",".join(categories),
        "filter": f"circle:{lon},{lat},{radius_meters}",
        "limit": 50,
        "apiKey": api_key,
    }
    async with httpx.AsyncClient(timeout=30) as client:
        r = await client.get(url, params=params)
        r.raise_for_status()
        return r.json()
```

### Normalization rules

* map API categories to internal `venue_type`
* extract website / address / coordinates when available
* store raw payload in `raw_places`
* create or update canonical `venues`

## Deduplication logic

Use a merge rule based on:

* normalized name similarity
* distance threshold
* same city/state

### Suggested logic

* lowercase
* strip punctuation
* remove stopwords like `the`, `museum of`, `center for`
* use RapidFuzz similarity
* if name similarity > 90 and distance < 200 meters, merge

### Pseudocode

```python
from rapidfuzz import fuzz


def is_same_venue(a, b):
    name_score = fuzz.token_sort_ratio(a["normalized_name"], b["normalized_name"])
    close_enough = a["distance_meters"] < 200
    return name_score > 90 and close_enough
```

## Image strategy

Images are not guaranteed from POI APIs. Use a multi-source approach.

Priority:

1. image from provider response if available
2. Wikimedia via wikidata/wikipedia
3. venue website hero image
4. manual/admin upload
5. category fallback image

Store images in `venue_images` and set `primary_image_url` on `venues`.

## Deterministic extraction layer

Before any LLM call, extract these with rules:

### Extractable fields

* title
* headings
* booking links
* price patterns
* duration patterns
* grade ranges
* capacity expressions
* structured data

### Regex examples

```python
PRICE_PATTERNS = [
    r"\$(\d+(?:\.\d{1,2})?)\s*(?:per student|/student|each)?",
]

DURATION_PATTERNS = [
    r"(\d+)\s*(minutes|minute|hours|hour)",
    r"half day",
    r"full day",
]

GRADE_PATTERNS = [
    r"grades?\s*(K|\d+)\s*[-–to]+\s*(\d+)",
    r"K[-–](\d+)",
    r"middle school",
    r"high school",
    r"all ages",
]
```

### Structured data extraction

Parse in this order:

* JSON-LD
* Microdata
* RDFa
* visible HTML

Capture fields from Schema.org entities when present.

## LLM extraction contract

### Input payload to model

Never send raw HTML. Send structured cleaned content.

```json
{
  "venue": {
    "name": "City Science Center",
    "website": "https://example.org"
  },
  "source": {
    "url": "https://example.org/education/field-trips",
    "page_type": "education_landing"
  },
  "content": {
    "title": "School Field Trips",
    "headings": [
      "Solar System Workshop",
      "Forces and Motion Lab"
    ],
    "text_blocks": [
      "Solar System Workshop: Students explore planets and gravity. Grades 3-6. 90 minute session. $12 per student.",
      "Forces and Motion Lab: Hands-on physics experiments for grades 4-8. Duration: 2 hours."
    ]
  },
  "deterministic_candidates": {
    "prices": ["$12 per student"],
    "durations": ["90 minute session", "2 hours"],
    "grade_ranges": ["Grades 3-6", "grades 4-8"]
  }
}
```

### Output schema

```json
{
  "programs": [
    {
      "program_title": "",
      "program_type": "",
      "grade_min": null,
      "grade_max": null,
      "duration_minutes": null,
      "price_per_student": null,
      "group_min": null,
      "group_max": null,
      "subjects": [],
      "indoor_outdoor": null,
      "booking_url": null,
      "confidence": 0.0,
      "evidence": {
        "title": "",
        "grade_range": "",
        "duration": "",
        "price": ""
      }
    }
  ]
}
```

### Rules for the model

* extract only information directly supported by the text
* never guess missing values
* return null when absent
* include evidence snippets
* return valid JSON only

### Program types enum

```python
PROGRAM_TYPES = [

    "lab_experience",

```

### Subject taxonomy

```python
SUBJECTS = [

    "engineering",
    "biology",
    "earth_science",
    "agriculture",
    "civics",
    "local_history",
    "career_exposure",

```

### Grade normalization rules

* `K` becomes `0`
* `K-2` becomes `0..2`
* `middle school` becomes `6..8`
* `high school` becomes `9..12`
* `all ages` becomes null/null

### Duration normalization rules

* `90 minutes` -> 90
* `2 hours` -> 120
* `half day` -> 180
* `full day` -> 360
* range like `2-3 hours` -> midpoint

## Validation layer

Never trust raw model output. Validate after extraction.

### Validation rules

* `grade_min <= grade_max`
* `grade_min` and `grade_max` between 0 and 12
* `duration_minutes` between 5 and 600
* `price_per_student >= 0`
* `program_type` must be in enum
* `subjects` must come from controlled list
* discard programs with `confidence < 0.4`

### Queue for review when

* confidence < 0.6
* price detected but ambiguous
* grade range missing
* unsupported venue category
* multiple conflicting extractions for same title

---

## Canonical merge logic for programs

The same program may appear on multiple pages.

Merge using:

* same `venue_id`
* normalized title similarity
* overlapping grade ranges
* similar durations

### Source precedence

1. `program_detail`
2. `pdf_brochure`
3. `booking`
4. `education_landing`
5. `general_visit`

## Ranking engine

The API should rank field trip options.

### Suggested score

```text
score =
  0.35 * distance_score
+ 0.20 * grade_fit_score
+ 0.15 * subject_match_score
+ 0.15 * logistics_score
+ 0.15 * trust_score
```

### Inputs

* distance from school
* grade overlap
* subject overlap
* image quality
* venue/program confidence
* website presence
* booking URL availability

### Distance scoring example

* 0 to 10 miles -> high
* 10 to 25 -> medium
* 25+ -> low

## Hard truth on execution

Do not try to make this a full marketplace in v1. The correct MVP is:

* good venue discovery
* solid program extraction
* auditable structured data
* fast ranked search

Everything else is distraction until the data layer works.
