# Migration Validation: Add Last Login Tracking

## Migration File
`20240305000005_add_last_login.sql`

## Purpose
Add last_login_at tracking to teachers table for activity monitoring.

## Changes Made

### Schema Changes
1. **Added Column**: `teachers.last_login_at` (TIMESTAMPTZ, nullable)
   - Tracks timestamp of last successful login
   - Nullable to support existing teachers who haven't logged in yet

2. **Added Index**: `idx_teachers_last_login`
   - Improves query performance for sorting/filtering by last login

## Validation Steps

### 1. Verify Column Exists
```sql
SELECT column_name, data_type, is_nullable
FROM information_schema.columns
WHERE table_name = 'teachers' 
  AND column_name = 'last_login_at';
```

**Expected Result:**
- Column exists with type `timestamp with time zone`
- `is_nullable = YES`

### 2. Verify Index Exists
```sql
SELECT indexname, indexdef
FROM pg_indexes
WHERE tablename = 'teachers' 
  AND indexname = 'idx_teachers_last_login';
```

**Expected Result:**
- Index exists on `last_login_at` column

### 3. Test Update Operation
```sql
-- Test updating last_login_at
UPDATE teachers 
SET last_login_at = NOW()
WHERE id = (SELECT id FROM teachers LIMIT 1);

-- Verify update
SELECT id, name, last_login_at
FROM teachers
WHERE last_login_at IS NOT NULL
LIMIT 5;
```

**Expected Result:**
- Update succeeds without errors
- Timestamp is properly stored

### 4. Test Query Performance
```sql
EXPLAIN ANALYZE
SELECT id, name, email, last_login_at
FROM teachers
ORDER BY last_login_at DESC NULLS LAST
LIMIT 20;
```

**Expected Result:**
- Query uses index for sorting
- Execution time < 10ms for tables with < 10k rows

## Rollback Plan

```sql
-- Remove index
DROP INDEX IF EXISTS idx_teachers_last_login;

-- Remove column
ALTER TABLE teachers DROP COLUMN IF EXISTS last_login_at;
```
