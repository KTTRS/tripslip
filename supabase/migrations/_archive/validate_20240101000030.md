## Tables Created

### venue_bookings
- **Purpose**: Stores venue booking information for trips
- **Key Fields**:
  - `id`: Primary key (UUID)
  - `trip_id`: Reference to trips table
  - `venue_id`: Reference to venues table
  - `experience_id`: Reference to experiences table
  - `scheduled_date`, `start_time`, `end_time`: Booking schedule
  - `student_count`, `chaperone_count`: Group size
  - `status`: Workflow status (pending, confirmed, modified, cancelled, completed)
  - `confirmation_number`: Unique booking identifier (format: VB-YYYYMMDD-XXXX)
  - `quoted_price_cents`, `deposit_cents`, `paid_cents`: Pricing information
  - `special_requirements`, `venue_notes`, `internal_notes`: Communication fields
  - Timestamps: `requested_at`, `confirmed_at`, `cancelled_at`, `completed_at`

### data_sharing_consents
- **Purpose**: Manages parent consent for sharing student data with venues
- **Key Fields**:
  - `id`: Primary key (UUID)
  - `student_id`: Reference to students table
  - `parent_id`: Reference to parents table
  - `booking_id`: Reference to venue_bookings table
  - `share_basic_info`: Consent for basic info (name, age, grade)
  - `share_medical_info`: Consent for medical information
  - `share_contact_info`: Consent for parent contact info
  - `share_emergency_info`: Consent for emergency contacts
  - `consented_at`, `revoked_at`: Consent timestamps
- **Constraints**: Unique constraint on (student_id, booking_id)

## Functions Created

### generate_booking_confirmation_number()
- **Purpose**: Generates unique booking confirmation numbers
- **Format**: VB-YYYYMMDD-XXXX (e.g., VB-20240115-A3F9)
- **Logic**:
  - Date part: Current date in YYYYMMDD format
  - Random part: 4-character uppercase alphanumeric string
  - Uniqueness check with retry logic (max 100 attempts)
- **Returns**: TEXT

### set_booking_confirmation_number()
- **Purpose**: Trigger function to auto-generate confirmation numbers
- **Trigger**: BEFORE INSERT on venue_bookings
- **Logic**: Sets confirmation_number if null or empty

## Notes

1. **Confirmation Number Format**: The VB-YYYYMMDD-XXXX format provides:
   - Easy identification (VB prefix for Venue Booking)
   - Date-based organization
   - Sufficient uniqueness with 4-character random suffix (1.6M combinations)

2. **Status Workflow**: The booking status follows this lifecycle:
   - `pending`: Initial state when booking is created
   - `confirmed`: Venue has confirmed the booking
   - `modified`: Booking details have been changed
   - `cancelled`: Booking has been cancelled
   - `completed`: Trip has been completed

3. **Data Sharing**: The consent system implements privacy-by-design:
   - Basic info sharing defaults to true (required for booking)
   - Medical and contact info default to false (opt-in)
   - Emergency info defaults to true (safety requirement)
   - Parents can revoke consent at any time

4. **RLS Security**: Policies ensure:
   - Teachers only see their own bookings
   - Venue employees only see bookings for their venue
   - Parents control their own consent records
   - No unauthorized data access
