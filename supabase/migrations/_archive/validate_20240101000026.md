great# Migration Validation: Search Functions

**Migration**: `20240101000026_create_search_functions.sql`  
**Requirements**: 3.2 (Geographic radius search), 3.1 (Text search), 3.8 (Search performance)

## Purpose

This migration creates PostgreSQL functions to support efficient venue search with:
- Geographic radius search using PostGIS
- Full-text search with relevance ranking
- Combined multi-criteria search

## Functions Created

### 1. `venues_within_radius`

Finds venues within a specified radius of a geographic point.

**Parameters**:
- `lat` (DOUBLE PRECISION): Latitude of search center
- `lng` (DOUBLE PRECISION): Longitude of search center
- `radius_meters` (DOUBLE PRECISION): Search radius in meters
- `venue_ids` (UUID[]): Optional array of venue IDs to filter

**Returns**: Table with venue IDs and distances in meters

**Performance**: Uses PostGIS `ST_DWithin` for efficient geographic queries with spatial indexes

### 2. `search_venues_by_text`

Performs full-text search across venue names and descriptions.

**Parameters**:
- `search_query` (TEXT): Search query (supports websearch syntax)
- `max_results` (INTEGER): Maximum number of results (default 100)

**Returns**: Table with venue IDs, names, descriptions, and relevance ranks

**Performance**: Uses PostgreSQL full-text search with GIN indexes

### 3. `search_venues`

Combined search function with multiple criteria.

**Parameters**:
- `search_text` (TEXT): Optional text search query
- `center_lat` (DOUBLE PRECISION): Optional search center latitude
- `center_lng` (DOUBLE PRECISION): Optional search center longitude
- `radius_miles` (DOUBLE PRECISION): Optional search radius in miles
- `min_capacity` (INTEGER): Optional minimum capacity filter
- `max_capacity` (INTEGER): Optional maximum capacity filter
- `verified_only` (BOOLEAN): Filter for verified venues only
- `max_results` (INTEGER): Maximum results (default 25)

**Returns**: Table with venue details, distances, and text ranks

**Performance**: Combines multiple filters efficiently on the database side

## Validation Steps

### 1. Verify PostGIS Extension

```sql
SELECT PostGIS_Version();
```

Expected: PostGIS version information

### 2. Test Geographic Search

```sql
-- Search for venues within 10 miles of a point (e.g., New York City)
SELECT * FROM venues_within_radius(
  40.7128,  -- lat
  -74.0060, -- lng
  16093.4,  -- 10 miles in meters
  NULL      -- all venues
);
```

Expected: Returns venues with distances, ordered by proximity

### 3. Test Text Search

```sql
-- Search for science museums
SELECT * FROM search_venues_by_text('science museum', 10);
```

Expected: Returns venues matching the search with relevance scores

### 4. Test Combined Search

```sql
-- Search for verified science museums within 25 miles of NYC
SELECT * FROM search_venues(
  'science museum',  -- search_text
  40.7128,          -- center_lat
  -74.0060,         -- center_lng
  25.0,             -- radius_miles
  NULL,             -- min_capacity
  NULL,             -- max_capacity
  TRUE,             -- verified_only
  10                -- max_results
);
```

Expected: Returns verified venues matching all criteria

### 5. Verify Permissions

```sql
-- Check that authenticated users can execute functions
SELECT has_function_privilege('authenticated', 'venues_within_radius(double precision, double precision, double precision, uuid[])', 'EXECUTE');
SELECT has_function_privilege('authenticated', 'search_venues_by_text(text, integer)', 'EXECUTE');
SELECT has_function_privilege('authenticated', 'search_venues(text, double precision, double precision, double precision, integer, integer, boolean, integer)', 'EXECUTE');
```

Expected: All return `true`

### 6. Performance Test

```sql
-- Test search performance (should complete in < 2 seconds for < 1000 venues)
EXPLAIN ANALYZE
SELECT * FROM search_venues(
  'museum',
  40.7128,
  -74.0060,
  50.0,
  NULL,
  NULL,
  FALSE,
  25
);
```

Expected: Execution time < 2000ms, uses indexes efficiently

## Rollback

If issues are found, drop the functions:

```sql
DROP FUNCTION IF EXISTS venues_within_radius;
DROP FUNCTION IF EXISTS search_venues_by_text;
DROP FUNCTION IF EXISTS search_venues;
```

## Dependencies

- Migration `20240101000020_extend_venues_table.sql` must be applied first
- PostGIS extension must be enabled
- Venues table must have `location` column with GIST index
- Venues table must have full-text search GIN index

## Notes

- Geographic searches use the WGS84 coordinate system (SRID 4326)
- Distances are calculated using PostGIS geography type for accuracy
- Text search uses English language configuration
- Functions are marked as STABLE for query optimization
- All functions support both authenticated and anonymous users for public search
