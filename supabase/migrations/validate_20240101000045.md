# Migration Validation: Create Notification Tables

**Migration File**: `20240101000045_create_notification_tables.sql`

**Purpose**: Create tables for notification preferences, email logs, and SMS logs to support the email and SMS notification Edge Functions.

## Tables Created

### 1. notification_preferences
Stores user preferences for email and SMS notifications.

**Columns**:
- `user_id` (UUID, PK, FK to auth.users)
- `email_enabled` (BOOLEAN, default true)
- `sms_enabled` (BOOLEAN, default false)
- `preferences` (JSONB, default '{}')
- `created_at` (TIMESTAMPTZ)
- `updated_at` (TIMESTAMPTZ)

**Indexes**:
- Primary key on `user_id`
- Index on `user_id` for faster lookups
- Partial index on `sms_enabled` for opt-in queries

**RLS Policies**:
- Users can view/update/insert their own preferences
- Service role can access all preferences

### 2. email_logs
Logs all email delivery attempts for audit and debugging.

**Columns**:
- `id` (UUID, PK)
- `to_email` (TEXT)
- `template_id` (TEXT)
- `status` (TEXT, CHECK: 'sent' or 'failed')
- `attempts` (INTEGER, default 1)
- `error_message` (TEXT, nullable)
- `sent_at` (TIMESTAMPTZ)
- `created_at` (TIMESTAMPTZ)

**Indexes**:
- Primary key on `id`
- Index on `to_email`
- Index on `status`
- Index on `sent_at` (DESC)
- Index on `template_id`

**RLS Policies**:
- Service role only access

### 3. sms_logs
Logs all SMS delivery attempts for audit and debugging.

**Columns**:
- `id` (UUID, PK)
- `to_phone` (TEXT)
- `message_preview` (TEXT, nullable)
- `status` (TEXT, CHECK: 'sent' or 'failed')
- `error_message` (TEXT, nullable)
- `twilio_message_id` (TEXT, nullable)
- `sent_at` (TIMESTAMPTZ)
- `created_at` (TIMESTAMPTZ)

**Indexes**:
- Primary key on `id`
- Index on `to_phone`
- Index on `status`
- Index on `sent_at` (DESC)
- Partial index on `twilio_message_id` (WHERE NOT NULL)

**RLS Policies**:
- Service role only access

## Notes

- The `sms_enabled` field defaults to `false` to ensure users explicitly opt in to SMS notifications (TCPA compliance)
- The `email_enabled` field defaults to `true` as email is the primary notification channel
- Email and SMS logs are service-role only to protect user privacy
- The `message_preview` in SMS logs is limited to 100 characters to save space
- Indexes are optimized for common query patterns (lookups by user, status, and date)
