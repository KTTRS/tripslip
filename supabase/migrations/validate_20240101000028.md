## Overview

This migration creates the venue categorization and tagging system, enabling hierarchical organization of venues through categories and flexible labeling through tags.

## Schema Changes

### New Tables

#### 1. `venue_categories`
Hierarchical categorization system for venues.

**Columns:**
- `id` (UUID, PK): Unique identifier
- `name` (TEXT, UNIQUE, NOT NULL): Category name
- `parent_id` (UUID, FK → venue_categories): Parent category for hierarchy
- `description` (TEXT): Category description
- `display_order` (INTEGER, DEFAULT 0): Sort order
- `created_at` (TIMESTAMPTZ, DEFAULT NOW())
- `updated_at` (TIMESTAMPTZ, DEFAULT NOW())

**Indexes:**
- `idx_venue_categories_parent`: On `parent_id` for hierarchical queries
- `idx_venue_categories_display_order`: On `display_order` for sorting

**Triggers:**
- `trigger_update_venue_categories_updated_at`: Auto-updates `updated_at` on modification

#### 2. `venue_category_assignments`
Many-to-many relationship between venues and categories.

**Columns:**
- `venue_id` (UUID, FK → venues, PK)
- `category_id` (UUID, FK → venue_categories, PK)
- `assigned_at` (TIMESTAMPTZ, DEFAULT NOW())

**Indexes:**
- `idx_venue_category_assignments_venue`: On `venue_id`
- `idx_venue_category_assignments_category`: On `category_id`

#### 3. `venue_tags`
Flat tagging system for venue features and themes.

**Columns:**
- `id` (UUID, PK): Unique identifier
- `name` (TEXT, UNIQUE, NOT NULL): Tag name
- `description` (TEXT): Tag description
- `created_at` (TIMESTAMPTZ, DEFAULT NOW())

**Indexes:**
- `idx_venue_tags_name`: On `name` for lookups

#### 4. `venue_tag_assignments`
Many-to-many relationship between venues and tags.

**Columns:**
- `venue_id` (UUID, FK → venues, PK)
- `tag_id` (UUID, FK → venue_tags, PK)
- `assigned_at` (TIMESTAMPTZ, DEFAULT NOW())

**Indexes:**
- `idx_venue_tag_assignments_venue`: On `venue_id`
- `idx_venue_tag_assignments_tag`: On `tag_id`

### New Functions

#### 1. `get_category_tree(category_id UUID)`
Recursively retrieves all subcategories of a given category.

**Returns:** TABLE (id UUID, name TEXT, parent_id UUID, level INTEGER)

**Usage:**
```sql
SELECT * FROM get_category_tree('category-uuid-here');
```

#### 2. `get_category_path(category_id UUID)`
Returns the full category path (breadcrumb) for a category.

**Returns:** TEXT (e.g., "Museums > Art Museums")

**Usage:**
```sql
SELECT get_category_path('category-uuid-here');
```

#### 3. `count_venues_in_category(category_id UUID)`
Counts venues in a category including all subcategories.

**Returns:** INTEGER

**Usage:**
```sql
SELECT count_venues_in_category('category-uuid-here');
```

## Performance Considerations

1. **Indexes:** All foreign keys and frequently queried columns are indexed
2. **Recursive Queries:** `get_category_tree()` uses CTE for efficient hierarchy traversal
3. **Caching:** Category tree should be cached in application layer (changes infrequently)
4. **Venue Counts:** Consider materializing venue counts for popular categories

## Security Considerations

1. **Public Read Access:** Categories and tags are publicly readable for discovery
2. **Admin-Only Creation:** Only admins can create/modify categories and tags
3. **Venue Employee Control:** Venue staff can only manage their own venue's assignments
4. **Cascade Deletes:** Deleting categories/tags removes all assignments automatically
