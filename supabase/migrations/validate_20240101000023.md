# Migration Validation: 20240101000023_create_venue_claim_requests

## Migration Summary
Creates the venue claim requests system including:
- `venue_claim_requests` table for tracking venue ownership claims
- Email verification workflow
- Admin review and approval workflow
- Automatic role assignment on approval
- RLS policies for security

## Requirements Addressed
- 5.1: Allow venue representatives to search and claim their venue
- 5.2: Require business email verification

- 5.5: Admin review and approval/rejection workflow

## Database Changes

### New Table: venue_claim_requests
```sql
CREATE TABLE venue_claim_requests (
  id UUID PRIMARY KEY,
  venue_id UUID REFERENCES venues(id),
  requester_id UUID REFERENCES auth.users(id),
  business_email TEXT NOT NULL,
  email_verified BOOLEAN DEFAULT false,
  email_verification_token TEXT,
  email_verification_sent_at TIMESTAMPTZ,
  proof_type TEXT CHECK (proof_type IN ('business_license', 'employment_verification', 'domain_email')),
  proof_document_url TEXT,
  status TEXT CHECK (status IN ('pending', 'under_review', 'approved', 'rejected')),
  reviewed_by UUID REFERENCES auth.users(id),
  reviewed_at TIMESTAMPTZ,
  rejection_reason TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### Indexes Created
- `idx_venue_claims_venue` - Fast lookup by venue
- `idx_venue_claims_requester` - Fast lookup by requester
- `idx_venue_claims_status` - Filter by status
- `idx_venue_claims_created` - Sort by creation date

### Triggers
1. **update_venue_claim_requests_updated_at** - Auto-update updated_at timestamp
2. **check_venue_claim_eligibility_trigger** - Prevent duplicate claims
3. **handle_venue_claim_approval_trigger** - Auto-grant roles on approval

### Functions
1. **check_venue_claim_eligibility()** - Validates claim eligibility:
   - Prevents claims for already claimed venues
   - Prevents duplicate pending/approved claims

2. **handle_venue_claim_approval()** - Handles claim approval:
   - Updates venue as claimed
   - Grants venue_admin role in user_role_assignments
   - Adds user to venue_users with administrator role

### RLS Policies
1. Users can view their own claim requests
2. Users can create claim requests
3. Users can update their own pending claims (for email verification)
4. TripSlip admins can view all claim requests
5. TripSlip admins can update claim requests (for review)
