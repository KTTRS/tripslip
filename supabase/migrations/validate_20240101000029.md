# Migration Validation: 20240101000029_create_venue_reviews

## Overview
This migration creates the venue review and rating system with multi-aspect ratings, venue responses, and moderation capabilities.

## Requirements Addressed
- 10.1: Teachers can submit reviews after trip completion
- 10.2: Reviews include overall rating (1-5 stars) and multi-aspect ratings
- 10.3: Reviews include text feedback (min 50 characters)
- 10.4: Reviews can include photos
- 10.5: Venues can respond to reviews
- 10.9: Reviews can be flagged for moderation
- 10.10: Flagged reviews are reviewed by TripSlip administrators

## Schema Changes

### New Tables

#### venue_reviews
- `id` (UUID, PK): Unique review identifier
- `venue_id` (UUID, FK → venues): Reference to reviewed venue
- `trip_id` (UUID, FK → trips): Reference to trip
- `user_id` (UUID, FK → auth.users): Review author
- `overall_rating` (INTEGER, 1-5): Overall rating (required)
- `educational_value_rating` (INTEGER, 1-5): Educational value rating (optional)
- `staff_quality_rating` (INTEGER, 1-5): Staff quality rating (optional)
- `facilities_rating` (INTEGER, 1-5): Facilities rating (optional)
- `value_rating` (INTEGER, 1-5): Value for money rating (optional)
- `feedback_text` (TEXT): Written feedback
- `photos` (TEXT[]): Array of photo URLs
- `venue_response` (TEXT): Venue's response to review
- `venue_response_at` (TIMESTAMPTZ): When venue responded
- `venue_response_by` (UUID, FK → auth.users): Who responded
- `flagged` (BOOLEAN): Whether review is flagged
- `flag_reason` (TEXT): Reason for flagging
- `reviewed_by_admin` (BOOLEAN): Whether admin has reviewed
- `admin_notes` (TEXT): Admin moderation notes
- `created_at` (TIMESTAMPTZ): Creation timestamp
- `updated_at` (TIMESTAMPTZ): Last update timestamp
- UNIQUE constraint on (venue_id, trip_id, user_id)

### Indexes Created
- `idx_venue_reviews_venue`: On venue_id (WHERE NOT flagged)
- `idx_venue_reviews_trip`: On trip_id
- `idx_venue_reviews_user`: On user_id
- `idx_venue_reviews_flagged`: On flagged (WHERE flagged = true)
- `idx_venue_reviews_created`: On created_at DESC

### Functions Created

#### update_venue_rating()
Trigger function that automatically updates venue.rating and venue.review_count when reviews are added, updated, or deleted. Only counts non-flagged reviews.

### Triggers Created
- `update_venue_rating_on_insert_update`: Fires AFTER INSERT OR UPDATE
- `update_venue_rating_on_delete`: Fires AFTER DELETE

### RLS Policies

#### Teachers
- Can view all non-flagged reviews (or their own flagged reviews)
- Can create reviews for their own trips
- Can update their own reviews
- Can delete their own reviews

#### Venue Employees
- Can view all reviews for their venue
- Can add responses to reviews for their venue

#### Admins
- Can view all reviews (including flagged)
- Can moderate reviews

## Performance Considerations

1. **Index on venue_id**: Ensures fast retrieval of all reviews for a venue
2. **Partial index on flagged**: Optimizes queries for flagged reviews
3. **Index on created_at**: Supports sorting by date efficiently
4. **Trigger efficiency**: Updates only the affected venue, not all venues

## Notes

1. The minimum feedback text length (50 characters) is enforced at the application level, not in the database
2. Photo URLs are stored as an array; actual photo storage is handled by Supabase Storage
3. Flagged reviews are excluded from rating calculations automatically
4. The trigger handles INSERT, UPDATE, and DELETE operations to keep ratings current
5. RLS policies ensure teachers can only review their own trips and venues can only respond to their own reviews
