# Migration Validation: 20240101000025_extend_experiences_table.sql

## Purpose
Extends the experiences table with additional fields required for comprehensive experience management, including educational objectives, curriculum standards, age ranges, cancellation policies, and active status.

## Requirements Addressed
- **Requirement 2.1**: Store experience name, description, educational objectives, and duration
- **Requirement 2.2**: Store pricing information including base price, per-student pricing, and group discounts
- **Requirement 2.3**: Store minimum and maximum group sizes
- **Requirement 2.4**: Store recommended age ranges and grade levels
- **Requirement 2.5**: Store subject area alignments and curriculum standards
- **Requirement 2.8**: Store cancellation policies and refund terms
- **Requirements 8.1-8.8**: Experience management operations

## Schema Changes

### Experiences Table Extensions

#### New Columns
1. **educational_objectives** (TEXT[])
   - Array of learning objectives for the experience
   - Default: empty array
   - Requirement: 2.1

2. **recommended_age_min** (INTEGER)
   - Minimum recommended age for participants
   - Nullable
   - Requirement: 2.4

3. **recommended_age_max** (INTEGER)
   - Maximum recommended age for participants
   - Nullable
   - Requirement: 2.4

4. **cancellation_policy** (JSONB)
   - Defines refund terms with structure:
     - fullRefundDays: Days before trip for full refund
     - partialRefundDays: Days before trip for partial refund
     - partialRefundPercent: Percentage refunded (0-100)
     - noRefundAfterDays: Days before trip when no refund available
   - Default: {fullRefundDays: 14, partialRefundDays: 7, partialRefundPercent: 50, noRefundAfterDays: 3}
   - Requirement: 2.8

5. **active** (BOOLEAN)
   - Whether experience is active and visible in search results
   - Default: true
   - Requirements: 8.6, 8.7

6. **special_requirements** (TEXT)
   - Any special requirements or notes for the experience
   - Nullable

#### Constraints
- **experiences_age_range_check**: Ensures age range is valid (min <= max) or both are null

### Pricing Tiers Table Extensions

#### New Columns
1. **additional_fees** (JSONB)
   - Array of additional fees with structure: [{name, amountCents, required}]
   - Default: empty array
   - Requirement: 2.2

### New Indexes
1. **idx_experiences_active**: Index on active column for search queries
2. **idx_experiences_published_active**: Composite index for published and active experiences
3. **idx_experiences_age_range**: Index on age range for filtering
