# Migration Validation: 20240101000020_extend_venues_table.sql

## Purpose
This migration extends the venues table to support comprehensive venue profiles with geographic search capabilities, profile claiming/verification, and search optimization.

## Requirements Addressed
- **1.1**: Store venue name, physical address, contact information, website URL, and description
- **1.2**: Store venue capacity ranges, supported age groups, relevant subject areas, and accessibility features
- **1.6**: Assign unique identifier (already exists in base table)
- **1.8**: Track venue profile completeness percentage
- **3.1**: Support text search across venue names and descriptions
- **3.2**: Filter venues by geographic location within a specified radius

## Changes Made

### New Columns Added
1. **website** (TEXT) - Venue website URL
2. **operating_hours** (JSONB) - Operating hours by day of week
3. **seasonal_availability** (JSONB) - Seasonal availability periods
4. **booking_lead_time_days** (INTEGER) - Minimum booking advance notice (default: 7)
5. **capacity_min** (INTEGER) - Minimum group capacity
6. **capacity_max** (INTEGER) - Maximum group capacity
7. **supported_age_groups** (TEXT[]) - Array of supported age groups
8. **accessibility_features** (JSONB) - Accessibility features object
9. **primary_photo_url** (TEXT) - Primary venue photo URL
10. **virtual_tour_url** (TEXT) - Virtual tour URL
11. **claimed** (BOOLEAN) - Profile claimed status (default: false)
12. **verified** (BOOLEAN) - Profile verified status (default: false)
13. **profile_completeness** (INTEGER) - Completion percentage 0-100 with CHECK constraint
14. **claimed_at** (TIMESTAMPTZ) - Claim timestamp
15. **verified_at** (TIMESTAMPTZ) - Verification timestamp
16. **rating** (DECIMAL(3,2)) - Average rating 0-5 with CHECK constraint
17. **review_count** (INTEGER) - Total reviews with CHECK constraint >= 0
18. **location** (GEOGRAPHY(POINT, 4326)) - PostGIS geographic point

### Extensions Enabled
- **postgis** - For geographic queries and spatial indexing

### Indexes Created
1. **idx_venues_location** (GIST) - Geographic radius searches
2. **idx_venues_search** (GIN) - Full-text search on name and description
3. **idx_venues_verified** - Filter verified venues
4. **idx_venues_claimed** - Filter claimed venues
5. **idx_venues_verified_rating** - Composite index for verified venues sorted by rating
6. **idx_venues_profile_completeness** - Profile completeness queries
7. **idx_venues_age_groups** (GIN) - Array search on age groups

### Data Migration
- Populates `location` column from existing `address` JSONB data (lng/lat)
- Only updates records where address data exists and location is NULL

  to_tsvector('english', name || ' ' || COALESCE(description, '')),
  to_tsquery('english', 'museum')

WHERE to_tsvector('english', name || ' ' || COALESCE(description, ''))

  ST_Distance(location, ST_MakePoint(-122.4194, 37.7749)::geography) / 1609.34 AS distance_miles

    location,
    ST_MakePoint(-122.4194, 37.7749)::geography,
    16093.4  -- 10 miles in meters
  )
ORDER BY distance_miles
LIMIT 10;

## Performance Considerations

1. **Geographic Queries**: GIST index on location column enables efficient radius searches
2. **Full-Text Search**: GIN index on tsvector enables fast text searches
3. **Partial Indexes**: Indexes on verified and claimed use WHERE clauses to reduce index size
4. **Array Search**: GIN index on supported_age_groups enables efficient array containment queries

## Security Considerations

1. All columns use `IF NOT EXISTS` to prevent errors on re-run
2. CHECK constraints prevent invalid data (rating 0-5, completeness 0-100)
3. Default values ensure data consistency
4. No sensitive data exposed in new columns
